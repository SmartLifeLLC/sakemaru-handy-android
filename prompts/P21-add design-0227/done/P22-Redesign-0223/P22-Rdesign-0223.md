```

-------------------|-----------------|-----------------|----------
화면 배경색        | <P21 값>        | <P22 값>        | 다르면 변경
카드 배경색        | <P21 값>        | <P22 값>        | 다르면 변경
카드 corner radius | <P21 값>        | <P22 값>        | 다르면 변경
카드 elevation     | <P21 값>        | <P22 값>        | 다르면 변경
카드 테두리색/두께 | <P21 값>        | <P22 값>        | 다르면 변경
텍스트 주색        | <P21 값>        | <P22 값>        | 다르면 변경
텍스트 보조색      | <P21 값>        | <P22 값>        | 다르면 변경
강조색             | <P21 값>        | <P22 값>        | 다르면 변경
버튼 스타일        | <P21 값>        | <P22 값>        | 다르면 변경
뱃지 스타일        | <P21 값>        | <P22 값>        | 다르면 변경

```

## 🎨 Step 2 — P22에 적용할 내용

Step 1 의 분석 결과를 바탕으로, **변경 필요** 항목만 P21의 값으로 교체한다.

### 적용 원칙

```
1. 헤더 영역          → 변경 금지 (기존 P22 헤더 유지)
2. 화면 배경색        → P21과 동일하게
3. 카드 스타일 전체   → P21의 corner radius / elevation / 테두리를 그대로 따름
4. 색상 체계          → P21의 배경·텍스트·강조·보조색 세트를 그대로 따름
5. 타이포그래피       → P21의 폰트 크기·굵기 체계를 그대로 따름
6. 간격·패딩          → P21의 카드 간격·내부 패딩 값을 그대로 따름
7. 버튼·뱃지          → P21에 존재하면 동일 스타일, P22에만 있으면 P21 색상 세트 적용
```

### 공통 색상 상수 처리

P21에서 사용 중인 색상 상수 파일 (예: `Colors.kt`, `Theme.kt`) 이 있다면:
- P22 도 동일 파일을 참조하도록 변경
- P22 전용 하드코딩 색상값이 있다면 공통 상수로 교체

```bash
# 색상 하드코딩 여부 확인
grep -n "Color(" <P22_파일경로>
grep -n "color=" <P22_XML파일경로>
```

## ✅ Step 4 — 완료 체크리스트

### 색상 통일

- [ ] P22 화면 배경색이 P21과 동일
- [ ] P22 카드 배경색이 P21과 동일
- [ ] P22 텍스트 주색·보조색이 P21과 동일

- [ ] P22 테두리색이 P21과 동일

### 컴포넌트 스타일 통일

- [ ] 카드 corner radius가 P21과 동일
- [ ] 카드 elevation/그림자가 P21과 동일
- [ ] 카드 내부 패딩이 P21과 동일
- [ ] 카드 간격이 P21과 동일
- [ ] 폰트 크기·굵기 체계가 P21과 동일
- [ ] 뱃지 스타일이 P21과 동일 (있는 경우)
- [ ] 버튼 스타일이 P21과 동일 (있는 경우)

### 유지 확인

- [ ] P22 헤더가 변경되지 않음
- [ ] 기존 이력 로직이 정상 동작
- [ ] 빌드 에러 없음
- [ ] 에뮬레이터에서 P21 / P22를 나란히 비교했을 때 색상·스타일이 통일되어 보임
