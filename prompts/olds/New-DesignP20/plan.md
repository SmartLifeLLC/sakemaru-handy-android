## P1: P21レイアウト変更

### 目的
P21（OutboundPickingScreen）のボディレイアウトを変更する。
- 右ペインの履歴リストを完全に削除
- 左ペインの商品情報はロケーション・伝票番号まで表示（現状維持）
- 受注数・出庫数の表示/入力欄を右ペインに移動
- 前へ/登録/次へボタンの配置を調整

### 変更前レイアウト（現状）
```
┌────────────────────────────────────────┐
│ TopAppBar (ヘッダー) ← 変更なし        │
├───────────────────┬────────────────────┤
│ 商品名/JAN/容量   │                    │
│ ロケーション/伝票  │    履歴リスト       │
│ 受注数 | 出庫数   │    (HistoryItemRow) │
│ [前へ][登録][次へ] │                    │
└───────────────────┴────────────────────┘
```

### 変更後レイアウト（目標）
```
┌────────────────────────────────────────┐
│ TopAppBar (ヘッダー) ← 変更なし        │
├───────────────────┬────────────────────┤
│ 商品名/JAN/容量   │ 受注数（読取専用）  │
│ ロケーション      │ 出庫数（入力欄）    │
│ 伝票番号          │                    │
│                   │ [前へ][登録][次へ]  │
└───────────────────┴────────────────────┘
```

### 修正対象ファイル
- `feature/outbound/src/main/java/biz/smt_life/android/feature/outbound/picking/OutboundPickingScreen.kt`

### 修正手順

1. **右ペイン（履歴）を削除**
   - `"履歴"` ラベルと履歴リストの `Surface` / `LazyColumn` / `HistoryItemRow` を削除
   - `HistoryItemRow` Composable 関数自体も削除（P22で使わないなら）

2. **左ペインの内容を調整**
   - 商品情報ヘッダー（商品名、JAN/容量/入数）をそのまま残す
   - ロケーション・伝票番号の表示行をそのまま残す
   - **受注数・出庫数セクションを左ペインから削除**
   - **前へ/登録/次へボタンを左ペインから削除**

3. **右ペインに受注数/出庫数+ボタンを配置**
   - 新しい右ペイン（weight 1f）を作成
   - 受注数（読取専用）セクションを配置
   - 出庫数（入力）セクションを配置
   - 前へ/登録/次へボタンを下部に配置

4. **全商品登録完了時の表示は維持**
   - 完了カード表示ロジックはそのまま維持

### 完了条件
- P21の右側に履歴リストが表示されないこと
- P21の左側に商品情報（ロケーション・伝票番号まで）が表示されること
- P21の右側に受注数・出庫数・ボタンが表示されること
- 全商品登録完了時の完了カード表示が正常動作すること

## P2: P22レイアウト変更

### 目的
P22（PickingHistoryScreen）のボディを、P21から削除した履歴リストのレイアウトで表示する。

### 変更方針
- 現在のP22のグリッドレイアウト（2列カード）を、P21の右ペインにあった履歴リスト形式に変更
- 各アイテム: 商品名、JAN/伝票番号、出庫数/受注数を表示
- 削除ボタン・確定ボタンの機能は維持
- 読取専用モード（全件確定済み）の表示は維持

### 変更前レイアウト（現状）
```
┌────────────────────────────────────────┐
│ TopAppBar (ヘッダー)                    │
├────────────────────────────────────────┤
│ ┌──────────┐  ┌──────────┐             │
│ │  カード1  │  │  カード2  │             │
│ │  2列グリッド                          │
│ └──────────┘  └──────────┘             │
│ ┌──────────┐  ┌──────────┐             │
│ │  カード3  │  │  カード4  │             │
│ └──────────┘  └──────────┘             │
├────────────────────────────────────────┤
│            [確定ボタン]                 │
└────────────────────────────────────────┘
```

### 変更後レイアウト（目標）
```
┌────────────────────────────────────────┐
│ TopAppBar (ヘッダー)                    │
├────────────────────────────────────────┤
│ 履歴リスト（LazyColumn）               │
│ ┌────────────────────────────────────┐ │
│ │ 商品名           [削除]            │ │
│ │ JAN / 伝票番号                     │ │
│ │ 出庫: X ケース / 受注: Y ケース    │ │
│ └────────────────────────────────────┘ │
│ ┌────────────────────────────────────┐ │
│ │ 商品名           [削除]            │ │
│ │ ...                                │ │
│ └────────────────────────────────────┘ │
├────────────────────────────────────────┤
│            [確定ボタン]                 │
└────────────────────────────────────────┘
```

### 修正対象ファイル
- `feature/outbound/src/main/java/biz/smt_life/android/feature/outbound/picking/PickingHistoryScreen.kt`

### 修正手順

1. **グリッドレイアウトをリストレイアウトに変更**
   - `LazyVerticalGrid(GridCells.Fixed(2))` を `LazyColumn` に変更
   - 各アイテムのカードレイアウトをリスト形式に変更

2. **各アイテムの表示内容**
   - 商品名（太字）+ 削除ボタン（行右端）
   - JAN コード or 伝票番号（小さいテキスト）
   - 「出庫: X ケース / 受注: Y ケース」形式の数量表示

3. **既存機能の維持**
   - 削除ダイアログ
   - 確定ダイアログ
   - 読取専用モード（全件確定済み）
   - 下部の確定ボタン

### 完了条件
- P22のボディが縦リスト形式で履歴を表示すること
- 各アイテムに商品名・JAN/伝票番号・数量が表示されること
- 削除・確定機能が正常に動作すること
- 読取専用モードが正常に動作すること
