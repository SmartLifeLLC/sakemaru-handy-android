## P1: 現状分析・共通スタイル定義

### 目的
HTMLリファレンスのデザインシステムをComposeのComposable/スタイルとして定義し、P2-P4で再利用可能にする

### 調査手順
1. 現行の各Screenファイルを読み込み、使用中のスタイル要素を把握
2. HTMLリファレンスのデザインシステムとの差分をリストアップ
3. 入庫処理（P10-P14）で既に実装済みの共通コンポーネントがあればそれを参照

### 作成する共通要素

#### A. カラー定数
```
bg-color: #f0f2f5        — 画面背景
text-color: #212529       — 通常テキスト
nav-bg: #1a233a          — ヘッダー背景
card-bg: #ffffff          — カード背景
btn-border-color: #6c757d — ボタン枠線
color-register: #2196f3   — 登録系(青)
color-history: #607d8b    — 履歴/戻る系(グレー)
color-select: #ff9800     — 選択系(オレンジ)
color-next: #4caf50       — 次へ(緑)
color-image: #9c27b0      — 画像(紫)
```

#### B. ヘッダー（Navbar）Composable
```
高さ: 60px
背景: #1a233a
構成: [左ボタン | 中央タイトル | 右ボタン]
ボタン: グラデーション(白→グレー)、枠線2px、下枠線5px、角丸8px
押下時: scale(0.95)、下枠線→2px、margin-top→3px
Fキーヒント: 0.75rem、色#444
```

#### C. フッターアクションバー Composable
```
背景: 白
上部ボーダー: 1px solid #ccc
パディング: 10px
ボタン: グラデーション(白→グレー)、枠線3px、下枠線8px(色は機能別)、角丸12px
押下時: scale(0.96)、下枠線→3px、margin-top→5px
```

#### D. 3Dボタン共通スタイル
- ヘッダー用（小さめ、下枠線5px）
- フッター用（大きめ、下枠線8px、機能別カラー）
- コースボタン用（フルワイド、下枠線8px）
- 削除ボタン用（小さめ、下枠線4px）

### 実装方針
- 入庫処理(P10-P14)で既に同様のスタイルが実装済みであれば、そのコンポーネントを共通化して再利用
- 未実装の場合は、出庫処理用の共通Composableファイルを作成

### 完了条件
- カラー定数が定義済み
- ヘッダー、フッター、ボタンのComposableが利用可能
- 入庫処理のデザインパターンとの一貫性が確認済み

## 制約（厳守）

- 画面解像度 1080x2400 / 420dpi / portrait・landscape対応 を前提にUI設計
- NoActionBar テーマ + Compose TopAppBar パターンを踏襲
- FunctionKeyBar の配置パターンを踏襲（出庫用にボタン数・色を変更）
- **ViewModel/Repository は変更しない**（UI層のみの変更）
- **API連携コードは変更しない**
- `local.properties` の認証情報はコミットしない
- サーバ側エラーは `error.log` に記録する
- HTMLリファレンスのデザインシステム（カラー、ボタンスタイル、レイアウト）に忠実に従う
- 入庫処理(P10-P14)のデザインと一貫性を保つ
- 3Dボタンの立体感（グラデーション + border-bottom）を統一
- Fキーヒントテキストを全ボタンに付与
