## 1. 背景

現在のピッキングシステムでは、各ピッカーは自分に割り当てられたタスクのみを処理します。
しかし、自分の作業が全て完了したピッカーが待機状態になるケースがあり、倉庫全体の効率が低下しています。

**実現したいこと:**
自分の作業が先に終わったピッカーが、まだ作業が残っている他のピッカーのタスクを「応援」（代行）できるようにしたい。

---

## 2. やりたいこと（3点）

### 2-1. 応援リストの取得

他のピッカーの未完了タスク一覧をハンディ端末で閲覧できるようにする。
「誰の」「どのコースの」タスクが「どの程度進んでいるか」を確認したい。

### 2-2. 未開始タスクの担当変更

まだ作業を開始していないタスク（全アイテムが PENDING）について、
応援者がそのタスクを「引き受ける」ことで担当者が変わり、応援者が通常通り作業を実施する。

### 2-3. 作業中タスクの部分応援（将来検討）

すでに作業中のタスク（一部アイテムが PICKING/COMPLETED）について、
残りのアイテムを応援者がリストの後方（walkingOrder の大きい方）からピッキングできるようにしたい。
→ ただし、これはシステム構造上の実現可能性を確認したい段階です。

---

## 3. 現在のAPI仕様（関連部分の整理）

### 現在のタスク取得API

```
GET /api/picking/tasks
  ?warehouse_id={id}    (必須)
  &picker_id={id}       (必須: 0で全タスク取得)
  &picking_area_id={id} (任意)
  &shipping_date={date} (任意)
```

**現在のレスポンス構造:**

```json
{
  "data": [
    {
      "course": { "code": "C001", "name": "コースA" },
      "picking_area": { "code": "A01", "name": "エリア1" },
      "wave": { "wms_picking_task_id": 123, "wms_wave_id": 456 },
      "picking_list": [
        {
          "wms_picking_item_result_id": 1,
          "item_id": 100,
          "item_name": "商品A",
          "status": "PENDING",
          "planned_qty": "3.00",
          "picked_qty": "0.00",
          "walking_order": 1,
          ...
        }
      ]
    }
  ]
}
```

**注意点:** 現在のレスポンスには **担当ピッカーの情報（picker_id, picker_name）が含まれていません。**

### 現在の更新系API

| エンドポイント | 用途 |
|---|---|
| `POST /api/picking/tasks/{id}/start` | タスク開始 |
| `POST /api/picking/tasks/{result_id}/update` | アイテム数量更新 |
| `POST /api/picking/tasks/{id}/complete` | タスク完了 |
| `POST /api/picking/tasks/{result_id}/cancel` | アイテムキャンセル |

## 4. API側で対応をお願いしたい事項

### 4-1. レスポンスへの担当者情報の追加

`GET /api/picking/tasks` のレスポンスに担当ピッカー情報を追加してほしい。

**追加希望フィールド（task単位、waveまたはトップレベル）:**

```json
{
  "course": { ... },
  "picking_area": { ... },
  "wave": { ... },
  "picker": {
    "id": 5,
    "code": "S001",
    "name": "山田太郎"
  },
  "picking_list": [ ... ]
}
```

**用途:** `picker_id=0` で全タスク取得した際に、各タスクが誰の担当かを表示するために必要。

### 4-2. 担当者変更APIの新設

未開始タスクの担当者を変更するエンドポイントが必要。

**想定仕様:**

```
POST /api/picking/tasks/{wms_picking_task_id}/reassign
```

**リクエストボディ:**
```json
{
  "new_picker_id": 10
}
```

**レスポンス:**
```json
{
  "data": {
    "wms_picking_task_id": 123,
    "previous_picker_id": 5,
    "new_picker_id": 10,
    "status": "OK"
  }
}
```

**バリデーション要件:**
- 全アイテムが PENDING のタスクのみ変更可能（作業中のタスクはエラーを返す）
- 既に完了済みのタスクはエラーを返す
