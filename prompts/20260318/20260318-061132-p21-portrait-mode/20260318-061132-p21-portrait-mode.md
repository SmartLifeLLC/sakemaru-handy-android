# P21 出庫データ入力画面 Portrait モード対応

- **作成日**: 2026-03-18
- **ステータス**: ドラフト
- **ディレクトリ**: `feature/outbound/src/main/java/biz/smt_life/android/feature/outbound/picking/`

## 背景・目的

P21（出庫データ入力画面）は現在 Landscape 固定で横2ペイン（左: 商品情報 / 右: 数量入力）のレイアウトになっている。
しかし、作業シーンによっては Portrait（縦向き）の方が操作しやすい場合がある。
ボタン操作で Landscape ⇔ Portrait を切り替え可能にし、Portrait 時は上部に商品情報・下部にピッキング数量入力を配置する縦レイアウトを提供する。

## 現状の実装

### 画面構成
- `OutboundPickingScreen.kt`（1018行）: メインUI。Scaffold + TopAppBar + Body
- `OutboundPickingBody`（340-692行）: Row で左右2ペインに分割
  - 左ペイン: 商品名、JAN、容量、顧客名、画像ボタン、ロケーション、伝票番号
  - 右ペイン: 数量入力（ケース/バラ）、登録ボタン、履歴ボタン
- `OutboundPickingViewModel.kt`（343行）: 状態管理
- `OutboundPickingState.kt`（105行）: UIステート

### 画面方向
- `AndroidManifest.xml` で `android:screenOrientation="landscape"` をアプリ全体に設定
- 画面内での向き切り替えロジックは未実装
- `MainActivity.kt` でイマーシブモード（ステータスバー/ナビバー非表示）を設定

## 変更内容

### 概要

P21 画面にのみ Portrait/Landscape 切り替えボタンを追加し、Portrait 時は縦2セクション（上: 商品情報 / 下: 数量入力）のレイアウトに切り替える。他画面は引き続き Landscape 固定。

### 詳細設計

#### 1. 画面方向制御の仕組み

**アプローチ**: `AndroidManifest.xml` の `screenOrientation="landscape"` はそのまま維持し、P21 画面内で `Activity.requestedOrientation` を動的に変更する。

- P21 に入った時: 現在の向き（デフォルト Landscape）を維持
- 切り替えボタン押下: `requestedOrientation` を `SCREEN_ORIENTATION_PORTRAIT` / `SCREEN_ORIENTATION_LANDSCAPE` にトグル
- P21 から離脱時: `requestedOrientation` を `SCREEN_ORIENTATION_LANDSCAPE` に戻す

```kotlin
// OutboundPickingScreen.kt 内
val context = LocalContext.current
val activity = context as? Activity

// 画面方向をStateで管理
var isPortrait by rememberSaveable { mutableStateOf(false) }

// 向き切り替え
fun toggleOrientation() {
    isPortrait = !isPortrait
    activity?.requestedOrientation = if (isPortrait) {
        ActivityInfo.SCREEN_ORIENTATION_PORTRAIT
    } else {
        ActivityInfo.SCREEN_ORIENTATION_LANDSCAPE
    }
}

// 画面離脱時にLandscapeに戻す
DisposableEffect(Unit) {
    onDispose {
        activity?.requestedOrientation = ActivityInfo.SCREEN_ORIENTATION_LANDSCAPE
    }
}
```

#### 2. 切り替えボタンの配置

TopAppBar の右側（既存ボタンの横）に回転アイコンボタンを追加する。

```kotlin
// TopAppBar actions に追加
IconButton(onClick = { toggleOrientation() }) {
    Icon(
        imageVector = Icons.Default.ScreenRotation, // または ScreenRotationAlt
        contentDescription = "画面回転",
        tint = AccentOrange
    )
}
```

**アイコン**: `Icons.Default.ScreenRotation`（Material Icons に含まれる）。なければカスタムアイコンを用意。

#### 3. Portrait レイアウト（縦向き時の UI）

Portrait 時は `OutboundPickingBody` 内で `Row`（横並び）を `Column`（縦並び）に切り替える。

**Portrait レイアウト構成**:

```
┌──────────────────────┐
│ TopAppBar            │ ← 回転ボタン含む
├──────────────────────┤
│ 【上部: 商品情報】     │
│  商品名               │
│  JAN / 容量 / 入数    │
│  顧客名               │
│  ロケーション          │
│  伝票番号             │
│  画像ボタン            │
├──────────────────────┤
│ 【下部: 数量入力】     │
│  数量入力フィールド     │
│  (ケース / バラ)       │
│  登録ボタン            │
│  履歴ボタン            │
└──────────────────────┘
```

**実装方針**:
- `isPortrait` フラグに基づいてレイアウト分岐
- 上部セクション: 現在の左ペインの内容をそのまま表示（横幅いっぱいに拡大）
- 下部セクション: 現在の右ペインの内容を表示
- 各セクションは `Modifier.weight(1f)` で縦方向に均等分割（または上部0.55 / 下部0.45 等の比率調整）
- 上部セクションは `verticalScroll` を維持して商品情報が多い場合にスクロール可能に

```kotlin
@Composable
fun OutboundPickingBody(
    // ... existing params ...
    isPortrait: Boolean,
) {
    if (isPortrait) {
        // 縦レイアウト
        Column(modifier = Modifier.fillMaxSize().padding(8.dp)) {
            // 上部: 商品情報
            Surface(
                modifier = Modifier.weight(0.55f).fillMaxWidth(),
                shape = RoundedCornerShape(12.dp),
                border = BorderStroke(1.dp, DividerGold),
            ) {
                ProductInfoSection(item, onShowImage)
            }
            Spacer(modifier = Modifier.height(8.dp))
            // 下部: 数量入力
            Surface(
                modifier = Modifier.weight(0.45f).fillMaxWidth(),
                shape = RoundedCornerShape(12.dp),
                border = BorderStroke(1.dp, DividerGold),
            ) {
                QuantityInputSection(...)
            }
        }
    } else {
        // 既存の横レイアウト（変更なし）
        Row(modifier = Modifier.fillMaxSize().padding(8.dp)) {
            // ... existing left/right pane code ...
        }
    }
}
```

#### 4. コンポーネント分離（リファクタリング）

現在 `OutboundPickingBody` 内に直接書かれている左ペイン・右ペインの内容を、再利用可能なコンポーザブルに分離する。

- `ProductInfoSection`: 商品情報セクション（現在の左ペイン内容）
- `QuantityInputSection`: 数量入力セクション（現在の右ペイン内容）

これにより Portrait / Landscape で同じコンポーネントを異なるレイアウトで配置可能。

#### 5. Portrait 時の UI 調整

- 商品情報セクション: 横幅が広くなるため、JAN/容量/入数を1行に収めやすくなる。レイアウトはそのまま流用。
- 数量入力セクション: 横幅フルなので入力フィールドとボタンを横並びにする余裕がある。ボタンは `Row` で横並びに配置。
- 画像ボタン: 商品情報セクション内で右上に配置を維持。
- フォントサイズ: Portrait/Landscape で変更なし（画面DPIが同じため）。

## 制約

1. **P21 限定**: Portrait 対応は P21 のみ。他画面は Landscape 固定を維持
2. **状態保持**: 向き切り替え時に入力中の数量や現在のアイテムインデックスが失われないこと（`rememberSaveable` で担保）
3. **イマーシブモード**: Portrait 時もステータスバー/ナビバーは非表示を維持
4. **離脱時復元**: P21 から戻る/ナビゲーション時に必ず Landscape に復元
5. **既存デザイン維持**: Landscape 時のレイアウトは現在の実装から変更しない
6. **ダイアログ互換**: ImageViewerDialog, CompletionConfirmationDialog は Portrait/Landscape 両方で正常表示すること
